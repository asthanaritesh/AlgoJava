An Adapter Pattern says that just "converts the interface of a class into another interface that a client wants".
In other words, to provide the interface according to client requirement while using the services of a class with a different interface.
The Adapter Pattern is also known as Wrapper.

Adapter pattern means using a different interface to implement its own functionality.
Assume, you can use two types of players: who can bat or who can batNBowl.
Let’s assume that you are short on players who can batNBowl and you would like to use batsman object in their place.
